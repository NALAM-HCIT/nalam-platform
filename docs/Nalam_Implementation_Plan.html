Nalam Backend — Implementation Plan (Admin Role)

A multi-tenant telemedicine backend for Nalam. Hospitals register via a web portal, and the registered mobile number becomes the admin. All authentication is mobile OTP only (via Pay4SMS). Admin manages users, settings, and dashboard from the React Native mobile app.

API Endpoints

Authentication

MethodEndpointDescription
POST/api/auth/send-otpSend OTP to mobile via Pay4SMS
POST/api/auth/verify-otpVerify OTP → return JWT (with role + hospital_id)
POST/api/auth/refreshRefresh expired access token

Hospital Registration (Public — from web portal)

MethodEndpointDescription
POST/api/hospitals/registerRegister hospital + auto-create admin user

Admin — User Management (Requires admin role)

MethodEndpointDescription
GET/api/admin/usersList users (search, filter by role/status)
POST/api/admin/usersCreate new user (doctor, pharmacist, etc.)
GET/api/admin/users/{id}Get single user details
PUT/api/admin/users/{id}Update user info
PATCH/api/admin/users/{id}/statusActivate / deactivate user
PATCH/api/admin/users/{id}/roleChange user role
DELETE/api/admin/users/{id}Remove user

Admin — Dashboard & Settings

MethodEndpointDescription
GET/api/admin/dashboardStats (total users, active staff, etc.)
GET/api/admin/activityRecent activity feed
GET/api/admin/settingsGet hospital settings
PUT/api/admin/settingsUpdate hospital settings
GET/api/admin/profileGet admin's own profile
PUT/api/admin/profileUpdate admin's own profile

Authentication Flow

sequenceDiagram participant App as Mobile App participant API as ASP.NET Core API participant SMS as Pay4SMS participant DB as Supabase PostgreSQL App->>API: POST /api/auth/send-otp {mobile: "+919876543210"} API->>DB: Find user by mobile_number alt User exists API->>API: Generate 6-digit OTP API->>DB: Store OTP (expires in 5 min) API->>SMS: Send OTP via Pay4SMS SMS-->>App: SMS delivered API-->>App: {success: true, message: "OTP sent"} else User not found API-->>App: {success: false, message: "Not registered"} end App->>API: POST /api/auth/verify-otp {mobile, otp} API->>DB: Validate OTP (not expired, not used) alt Valid API->>DB: Mark OTP as used, update last_login API-->>App: {token, refreshToken, user: {name, role, hospitalId}} else Invalid API-->>App: {success: false, message: "Invalid OTP"} end

Proposed Changes

  • Full project with Entity Framework Core + Npgsql for Supabase PostgreSQL
  • JWT authentication with role-based authorization
  • Pay4SMS OTP service integration
  • Multi-tenant data isolation via hospital_id in JWT claims
  • Dockerfile for Render deployment

  • login.tsx — Remove Employee ID login, keep OTP only, connect to real API
  • otp.tsx — Connect to verify-otp endpoint, store JWT in secure store
  • authStore.ts — Add token, refreshToken, hospitalId to state
  • Dashboard, Users, Settings, Profile — Replace mock data with real API calls

NEW services/api.ts

  • Axios instance with JWT interceptor & base URL config
  • Auto-refresh token on 401 responses
